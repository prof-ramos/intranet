# Issues Encontrados — Revisão 2026-03-18

## Resumo da Revisão Sistemática

Durante a etapa "Revisão Sistemática" (Etapa 4/5), foram aplicadas as ferramentas **Laravel Pint** e **Larastan (PHPStan)** para identificar falhas remanescentes de segurança, código anêmico, N+1 queries ou vazamentos no UI após a geração inicial de rotas e modelos.

## Críticos (bloqueiam lançamento)

Nenhum issue crítico classificado como bloqueador (e.g., Auth Bypass explícito) foi encontrado.

## Altos (devem ser corrigidos)

- [x] **#1 [PERF] NoticeRepository - N+1 Query Detected**
  - **Arquivo**: `app/Repositories/NoticeRepository.php:25`
  - **Problema**: O método `getActiveAndPublished()` varria notices individualmente e geraria centenas de queries quando percorrido pelo JSON via Lazy Load em `$notice->createdBy`.
  - **Correção**: Implementado `with('createdBy')` no pipeline da Query Builder.

- [x] **#2 [PERF] TaskHistoryRepository - N+1 Query Detected**
  - **Arquivo**: `app/Repositories/TaskHistoryRepository.php:12`
  - **Problema**: Método `getByTaskId()` não fazia eagerly load do objeto User ou da Tarefa.
  - **Correção**: Adicionada a instrução `with(['user', 'task'])` encadeada no Eager Load.

## Médios (melhorias)

- [x] **#3 [CODE] VerifyEmailController Falha de Sintaxe do PHPStan**
  - **Arquivo**: `app/Http/Controllers/Auth/VerifyEmailController.php:22`
  - **Problema**: O PHP não conseguia garantir ao evento `Verified` que o Request User obedeceria à inferência estrita exigida pelo `MustVerifyEmail`.
  - **Correção**: Adicição de DocBlock assertivo `/** @var \App\Models\User $user */` e extração de variável explícita.

- [x] **#4 [CODE] Padrões não encontrados do User Model via Larastan**
  - **Arquivo**: `app/Models/User.php:13-14`
  - **Problema**: Os pseudo-attributes anotados via `#[Fillable]` e `#[Hidden]` não são processados nativamente pelo boilerplate raiz do Laravel 11.
  - **Correção**: Convertido para o formato canônico `protected $fillable = [...]` e imposta à classe a implementação de `MustVerifyEmail`.

- [x] **#5 [CODE] JsonResource Mixins omitidos**
  - **Arquivo**: Todos os 5 resources gerados (e.g. `ContactResource.php`).
  - **Problema**: Arquivos apontavam atributos mágicos dinâmicos (`$this->id`) gerando até 43 throw errors no PHPStan.
  - **Correção**: Adição de anotações PHPDoc `@mixin \App\Models\EntityBase` no pre-ambulo.

## Baixos (nice to have)

- [x] **#6 [STYLE] Extensa inconsistência de Code Style em 61 Arquivos**
  - **Problema**: Diversos espaços, quebras de linhas de atributos mágicos, espaçamento em namespaces no scaffolding padrão.
  - **Correção**: O comando `./vendor/bin/pint` corrigiu automaticamente 22 style issues através do projeto base.

- [x] **#7 [UI] AlpineJS flickering de componentes (NoticeCard)**
  - **Problema**: Variáveis como `x-show="open"` poderiam exibir conteúdo precocemente devido ao tempo de bootstrap.
  - **Correção**: A propriedade CSS `[x-cloak] { display: none !important; }` foi emparelhada globalmente no `app.css`.
